# 学习日历主页 —— 设计稿（2026-06-02）

## 背景

welcome 问卷第 4 步「你有想做的产品 idea 吗？」是分流闸门。当前两条线最终都落到 `roadmap.html`，但 roadmap 是围绕「你的项目」的（5 个项目阶段 + 知识地图）。对选「还没有，先跟课程做」的学员，他还没项目，roadmap 的项目引擎给他塞一个占位的默认教学项目，基本空转。

读过课程内容（两份飞书文档）后确认一个关键点：**「没 idea」≠「没项目」**——课程是拿默认教学项目（黑话翻译器 → 哄哄模拟器 → 纸片人男友，纸片人男友贯穿整个进阶）从头带到上线。所以「跟课走」的人真正需要的是一条「今天学哪节、做到哪一步」的时间线，而不是项目管理面板。

## 决策（已与产品确认）

1. **路由对称**：
   - 有 idea → diagnosis（刘小排诊断）→ `roadmap.html`（项目为中心）。**不变。**
   - 没 idea → `calendar.html`（课程主线为中心）。**新方向。**
   - 两条线都走完整问卷（第 5 步节奏 + 第 6 步加微信），只在第 6 步那张卡片分头：没 idea 卡片显示「查看我的学习日历 →」去 calendar；有 idea 显示「查看我的学习路线 →」去 roadmap。
2. **日历当主场**：日历是「没 idea」学员的日常主页；roadmap 退成「偶尔看全局 / 知识地图」的副页（从日历顶栏可达）。
3. **主按钮 = 继续学习 · 下一节**：按 `ds_completed_demo`（已学课程 code 数组）算出下一节，一键进 `learn.html?code=`。
4. **旋钮默认收起**：目标 / 每天投入 / 学习日 / 开始日期默认隐藏，点「调整计划 ▾」才展开。
5. **课表**：已学的打勾标灰，「下一节」高亮。
6. **加刘小白浮窗**：引 `assistant.js`。

## 日历主页布局（从上到下）

1. **顶栏**（新）：logo + 「已学 X/77」进度条 + 「路线图/知识地图 →」出口 + 头像。
2. **主按钮区**（新）：「继续学习 · 下一节：XX」+「开始学习 →」+ 副行「已学 X/77 · 约 X 月 X 日学完 · 当前阶段」。
3. **调整计划**（改）：折叠的旋钮面板，默认收起。
4. **概要 + 课表**（改）：保留概要 4 格；课表已学打勾标灰、下一节高亮。
5. **刘小白浮窗**（新）。

参考草图：`prototype/calendar-home-mockup.html`（临时件，定稿后删）。

## 涉及数据契约（沿用，不新增 key）

- `ds_has_idea`（welcome 写）：决定第 6 步卡片去向。
- `ds_daily_minutes` / `ds_enrolled_at`（welcome 写）：日历排课节奏 + 起始日。**已接入。**
- `ds_completed_demo`（learn/roadmap 读写）：算「已学 / 下一节」。
- `COURSES`（course-data.js）：课程数据源（含 code/stage/mins/t）。

## 文件改动

- `welcome.html`：第 6 步卡片的 href + 文案按 `ds_has_idea` 切换。
- `calendar.html`：加顶栏 + 主按钮（含「下一节」计算）+ 旋钮折叠 + 课表打勾/高亮 + 引 assistant.js。

## 分两步走

- **第一步（本设计）**：路由 + 日历主页化。**不依赖课程数据是否准确。**
- **第二步（待课程数据核对后）**：在课表里标教学项目里程碑（「第 X 天做出黑话翻译器」等）。前提是先把 `course-data.js` 的 77 节与两份飞书文档逐模块核对（尤其 D 阶段新出现的【内功】模块在现有四阶段模型里没有位置）。
