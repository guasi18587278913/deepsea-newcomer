# 新手引导重设计 · 设计稿

- 日期：2026-06-16
- 范围：学习地图（calendar.html）首访新人引导（现 `#sctour` 那串气泡）
- 状态：已与产品对齐，待实现

## 1. 为什么重做

现在的引导是"手指点一下 + 一段话"——属于**告诉**用户界面长啥样（这是地图、你在这、船会升级、菜单在顶上…）。用户看完记不住重点，也不知道**要学会什么**。

新方向：**演给用户看**，而不是说。每一条信息都用动画把"行为"演一遍。

## 2. 要传达的 5 件事（只演这些，不加任何多余文字）

1. 每座岛 = 一个学习阶段
2. 点开岛 = 看这阶段的内容：图文 / 视频 / 作业
3. 图文全部看完 = 这阶段通关（视频、作业不计——与现有"只图文计解锁"一致）
4. 通关后，可以换一条新路线
5. 右下角浮窗：随时跳回正在学的内容 + 看最新直播动态

## 3. 形式（已定）

- **分步演 · 每步自动动**：仍分 5 步，保留「跳过 / 上一步 / 下一步（末步"完成"）」+ 进度点；但每步不再是静态文字，而是**自动播放一段演示动画**。用户可控制节奏。
- **不要刘小排**：去掉"刘小排 / 你的领航员"头像与人设。纯字幕 + 动画。
- **字幕只有这 5 句**，其余现有文案（"欢迎登船船长""卡住了去广场喊一声""工具箱""船会升级"…）全部删除。

## 4. 分镜（5 步）

| 步 | 高亮目标 | 演什么动画 | 字幕（屏上唯一文字） |
|---|---|---|---|
| ① 岛=阶段 | 全部 4 座岛 | 4 座岛由近到远依次轻亮，各浮出"阶段 1/2/3/4" | 每座岛 = 一个学习阶段 |
| ② 点开看内容 | 课程抽屉 `#kmCallout` | 自动点开第一座岛 → 抽屉滑出 → 图文/视频/作业三类各扫亮一下 | 点开岛，看这阶段的内容：图文、视频、作业 |
| ③ 图文看完才通关 | 抽屉里的图文项 → 当前岛 | 图文项逐个自动打勾 → 打满后该岛通关点亮 + 下一座岛解锁（开锁动画） | 图文全部看完，这一阶段才通关 |
| ④ 通关后换路线 | 「找到适合你的航线」按钮 | 紧接通关，高亮该按钮并轻跳一下 | 通关后，可以在这儿换一条新路线 |
| ⑤ 右下角浮窗 | 右下角浮窗 | 高亮浮窗 → 演示"跳回在学内容" + 弹出一条直播动态（红条闪一下） | 右下角浮窗：随时跳回在学内容，还能看最新直播 |

## 5. 依赖：右下角浮窗要补到默认地图

- 现状：「继续学习 + 直播动态」右下角浮窗**只在专属航线模式有**（route 模式下复用并重定位的卡）。默认全量地图上没有——默认图的"继续学习"是插在岛边的小木牌，直播信息藏在一个平时隐藏的卡里。
- 决定：**默认地图也加上这个右下角浮窗**，两种模式统一。引导第 ⑤ 步因此在两种模式都能演。
- 待实现时定的小点（建议先按推荐做）：
  - **推荐**：新浮窗**取代**岛边的"继续学习"小木牌，避免出现两个"继续学习"入口。
  - 浮窗内容 = 继续学习（跳回在学课程）+ 一条"正在直播/最新直播"动态。

## 6. 约束与注意

- **不污染真实进度**：③④ 的打勾 / 通关 / 解锁是**演示用的假动作**，只做视觉动画，**不得**调用真实的 `toggleCourse` / 写 `ds_completed_demo` / `ds_ship_level`。
- **两种模式都要能跑**：目标元素（岛、抽屉、找航线按钮、右下角浮窗）在默认图和专属航线下都要能解析到；解析不到时安全跳过该步，不要卡死或乱跳。
- **首访自动弹 + 可重看**：保留"看过一次不再自动弹"（`ds_tour_done`）与右上角"↻ 新手引导"重看入口。

## 7. 受影响文件

- `calendar.html`：重写 `#sctour` 那段自包含引导（HTML 气泡结构 + STEPS + 动画脚本）。
- 右下角浮窗：把 route 模式的浮窗实现（routes-ui.js / route-art-apply.js 里那套）抽出/复用到默认地图；具体落点实现时定。

## 8. 不做（YAGNI）

- 不加"船会升级""顶栏导航""广场/工具箱"等任何 5 件事以外的讲解。
- 不做自动一镜到底的播片（已选分步形式）。
- 不引入刘小排人设。
