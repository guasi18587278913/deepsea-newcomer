# 优化思路备忘 · 学习地图页信息密度

> 来源：2026-06-17 会议后定的优化点之一「压信息密度」。
> 目标：中间大动画占了约 3/4 屏，真正能点去学的内容挤成一小条 —— 压掉无效区、把重点提上来。
> 文件：`calendar.html`（页面标题「你的学习地图」）

## 现状问题（从上到下拆屏看）

1. **头部太占地方**
   - `.km-head .plan-title` 字号被设成 **54px**（CSS 约 line 321），下面还有 `.km-head-sub` 副标题框 + `#homeHero` 一段 hero。
   - 顶栏（56px）+ 这个头，吃掉差不多**屏幕上面 1/3**。

2. **中间全是"看的"，不是"点的"**
   - 大海图容器 `.km-wrap#kmWrap` 高度写死 `calc(100vh - 316px)`、`max-height:640px`（约占 6 成屏）。
   - 里面 `.km-gulls`（海鸥）`.km-bottle`（漂流瓶）`.km-compass`（指南针）`.km-ship`（帆船）`.km-route`（航线）**全是装饰，`pointer-events:none`，点不了**。

3. **真正的课程清单被藏起来了（最要命）**
   - 能点去学的那串课（图文/视频/作业 = `kd-flow` / `kd-flow-item`）在 `#kmCallout` 浮层里，**点了某个 `.island` 岛屿才弹出来**。
   - 所以第一眼：满屏大标题 + 装饰海洋，"我现在该点啥学"反而看不到。

## 优化方向（按性价比排）

- **① 先砍头部（最快见效）**：`plan-title` 54px → 约 30px，`#homeHero` 收成一条窄的。光这步腾出约 100~150px，重点立刻上移。
- **③ 把"下一步该学啥"提到首屏常驻（最值钱）**：做一张固定的小卡/一条——"你现在该学这个 →"，别全塞在"点岛屿才弹"的 `kmCallout` 浮层里。这就是"把重点提上来"的核心。
- **② 压装饰、放大可点**：`.km-wrap` 最大高度 640 → 约 480，海鸥/帆船/指南针等装饰弱化；让 `.island` 岛屿更大、更像按钮、名字常显。

## 建议落地顺序

先做 ① + ③（动头部 + 首屏常驻下一步），做完起本地预览看效果，再决定 ② 压多少。
保留大脑钩子不要动：`.wrap` / `#kmWrap` / `.km-wrap .island` / `.km-dot` / `#kmShip` / `#homeHero` / `#kmCallout`。
